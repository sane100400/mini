기존 타임테이블의 문제는 너무 넓었다. 애드센스, 숏폼, 전자책, 상세페이지를 한꺼번에 굴리려고 하니 글은 많아도 검색할 이유가 약했다. "AI 부업을 해보자"는 말은 누구나 한다. 방문자가 궁금한 건 Outlier가 실제로 일감이 있는지, DataAnnotation 코딩 일은 어떤 사람에게 맞는지, 바이브 코딩으로 돈을 받으려면 무엇을 팔아야 하는지다.

그래서 30일 운영표를 다시 짰다. 기준은 퇴근 후 2시간이다. 이번에는 글감 수가 아니라 질문의 날카로움을 기준으로 둔다. 큰 주제 하나를 잡고, 그 안에서 사람들이 검색할 법한 질문을 끝까지 쪼갠다.

![책상 위에 놓인 플래너와 노트](./images/calendar-planning.jpg)

## 1주 차는 플랫폼 비교로 시작한다

처음 7일은 Outlier 같은 해외 AI 부업 플랫폼을 정리한다. 단순 소개가 아니라 가입 전 확인할 것, 한국인이 막히는 지점, 코딩 가능자에게 유리한 플랫폼을 나눈다.

| 날짜 | 작업 | 산출물 |
|---|---|---|
| 1일차 | Outlier 공식 화면 캡처와 모집 상태 확인 | 캡처 1장, 메모 |
| 2일차 | DataAnnotation 코딩 페이지 확인 | 코딩 부업 메모 |
| 3일차 | Alignerr, Mindrift 비교 | 플랫폼 표 |
| 4일차 | 한국인이 막히는 지점 정리 | 영어·심사·결제 체크리스트 |
| 5일차 | 플랫폼별 지원 전 준비물 작성 | 샘플 준비표 |
| 6일차 | 플랫폼 비교 글 발행 | 긴 글 1개 |
| 7일차 | 관련 짧은 글 제목 5개 뽑기 | 후속 글 목록 |

이 주에는 "추천합니다"로 끝내지 않는다. 공식 화면, 단가 문구, 공개 기회 수, 심사 방식처럼 확인 가능한 것을 넣는다. 그래야 글이 검색 결과에서 버틴다.

![노트북과 노트에 아이디어를 적는 작업 공간](./images/notes-laptop.jpg)

## 2주 차는 바이브 코딩을 파고든다

8일차부터는 바이브 코딩을 다룬다. 여기서도 "AI로 앱 만들기"처럼 넓은 글은 버린다. 사람들이 돈을 낼 만한 작은 납품물을 기준으로 쓴다.

| 날짜 | 작업 | 산출물 |
|---|---|---|
| 8일차 | v0, Cursor 공개 화면 캡처 | 도구 이미지 |
| 9일차 | 팔기 좋은 납품물 5개 정리 | 상품 표 |
| 10일차 | 팔면 안 되는 작업 정리 | 리스크 표 |
| 11일차 | 전자책 판매 페이지 샘플 기획 | 샘플 구조 |
| 12일차 | 가격 계산기 샘플 기획 | 미니툴 구조 |
| 13일차 | 바이브 코딩 글 발행 | 긴 글 1개 |
| 14일차 | 크몽·블로그 문의용 상품 문구 초안 | 판매 설명 |

바이브 코딩 글에는 반드시 검수 기준을 넣는다. 모바일에서 깨지는지, 폼이 실제로 도착하는지, 배포 링크가 열리는지 같은 내용이 있어야 한다. 도구 이름만 나열하면 또 얕아진다.

## 3주 차는 플랫폼 글을 후속 글로 쪼갠다

15일차부터는 큰 글에서 파생 글을 만든다. 예를 들어 플랫폼 비교 글 하나에서 아래 글감이 나온다.

![집에서 노트북으로 작업하는 재택 업무 화면](./images/home-office.jpg)

- Outlier 지원 전에 준비할 영문 자기소개
- DataAnnotation 코딩 테스트 전에 볼 GitHub 샘플
- Alignerr AI 인터뷰에서 말할 전공 경험 정리
- Mindrift 초급형과 전문가형 차이
- 해외 AI 부업 플랫폼에서 한국어 프로젝트만 기다리면 안 되는 이유

이런 글은 서로 내부 링크를 걸기 쉽다. 한 글을 읽은 사람이 다음 글로 넘어갈 이유도 생긴다.

## 4주 차는 숫자보다 질문을 본다

마지막 7일은 조회수만 보지 않는다. 어떤 질문이 다음 글로 이어졌는지를 본다. 검색 유입이 적어도 댓글, 문의, 체류 시간이 길면 남길 만하다. 반대로 글 제목은 좋아 보여도 다음 행동이 없으면 버린다.

이번 달의 기준은 단순하다.

- 플랫폼 비교 글 3개
- 바이브 코딩 글 3개
- 전자책·블로그 연결 글 2개
- 실제 캡처 또는 샘플 이미지가 들어간 글 5개
- 내부 링크로 이어지는 글 묶음 2개

이제 블로그는 "AI 부업 추천"을 넓게 말하지 않는다. Outlier를 검색한 사람, 바이브 코딩으로 외주를 받고 싶은 사람, 코딩 부업 플랫폼이 궁금한 사람처럼 더 좁은 독자를 잡는다. 글감이 줄어드는 대신 글의 이유가 생긴다.

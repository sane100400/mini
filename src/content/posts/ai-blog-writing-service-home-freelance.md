AI 블로그 원고 대행을 부업으로 생각할 때 제일 먼저 버려야 할 생각이 있다. ChatGPT로 글을 뽑아서 그대로 파는 방식이다. 그 정도는 의뢰인도 직접 한다. 프리랜서가 돈을 받으려면 AI 초안이 아니라 **업종을 읽고, 목차를 잡고, 납품 가능한 글로 고치는 과정**을 팔아야 한다.

재택 진행이 가능한 점은 장점이다. 노트북과 자료만 있으면 시작하고, 통화 없이 메시지로 진행하기도 쉽다. 대신 경쟁도 많다. "블로그 원고 써드립니다"라는 말만으로는 차이가 안 난다. 어떤 자료를 받고, 어떤 범위까지 고치고, 어떤 결과물로 납품하는지가 보여야 한다. ✍️

![노트에 원고 구조를 적어두고 작업하는 장면](./images/writing-desk.jpg)

_참고자료 사진: 원고 대행은 글자 수보다 구조와 수정 범위를 먼저 잡아야 한다._

> ✍️ **처음 팔 때 핵심**
> - AI 초안 자체가 아니라 자료 정리와 수정 과정을 판다.
> - 제목 후보, 목차, 본문 초안, 수정 횟수를 미리 정한다.
> - 사실 확인이 필요한 부분은 의뢰인 확인으로 남겨둔다.

| 팔아야 하는 것 | 팔면 약한 것 |
| --- | --- |
| 업종 자료를 읽고 목차를 잡는 과정 | AI가 만든 초안 그대로 |
| 제목 후보와 본문 구조 | 글자 수만 많은 원고 |
| 수정 범위가 정해진 납품물 | 수정 무제한 상품 |
| 사실 확인이 필요한 부분 표시 | 그럴듯한 일반론 |

## 🛒 마켓 화면부터 본다

블로그 원고 대행을 팔려면 실제 마켓에서 이 일이 어떤 이름으로 팔리는지 먼저 봐야 한다. 아래는 크몽 공개 메인 화면이다. 마케팅, 전자책, IT·프로그래밍, AI 서비스 카테고리가 같이 보인다. 원고 대행은 이 중 하나에만 갇히지 않는다.

![크몽 공개 메인 화면. 마케팅, 전자책, AI 서비스 카테고리가 함께 보인다.](./images/kmong-marketplace-screen.jpg)

_참고자료 사진: 마켓 화면을 보면 의뢰인이 어떤 이름으로 이 일을 찾는지 감이 온다._

| 의뢰인이 찾는 이름 | 실제 필요한 작업 |
| --- | --- |
| 블로그 원고 | 업종 조사, 제목, 목차, 본문 |
| 체험단 글 | 방문 경험 정리, 후기 문체 조정 |
| 전자책 소개글 | 대상 독자, 목차, 구매 이유 정리 |
| 상세페이지 문구 | 고객 질문, 짧은 판매 문장 |
| AI 원고 | 초안 생성보다 검수와 수정 |

마켓을 먼저 보면 상품 설명도 현실적으로 바뀐다. "AI로 빠르게 써드립니다"보다 "업종 자료를 보고 제목 3개, 목차 1개, 본문 초안을 납품합니다"가 낫다. "수정 무제한"보다 "초안 1회, 문장 수정 1회, 사실관계는 의뢰인 확인"처럼 적는 편이 안전하다.

## 📦 최소 상품은 원고 한 편이 아니다

초보가 팔기 좋은 최소 단위는 원고 한 편처럼 보인다. 하지만 원고 한 편만 팔면 수정 요청이 끝없이 늘어난다. 의뢰인은 제목도 바꾸고 싶어 하고, 문체도 바꾸고 싶어 하고, 업종 정보도 더 넣고 싶어 한다. 그래서 최소 상품은 글자 수가 아니라 작업 범위로 잡아야 한다.

| 기본 상품 구성 | 포함 여부 |
| --- | --- |
| 업종 자료 확인 30분 | 포함 |
| 제목 후보 3개 | 포함 |
| 목차 1개 | 포함 |
| 본문 초안 1개 | 포함 |
| 문장 수정 1회 | 포함 |
| SEO 키워드 조사 | 별도 옵션 |
| 이미지 선정/업로드 | 별도 옵션 |
| 인터뷰 정리 | 별도 옵션 |

![노트북으로 원고를 검토하는 책상](./images/content-desk.jpg)

_참고자료 사진: 원고를 많이 쓰는 것보다 납품 전 검수 기준을 만드는 쪽이 더 중요하다._

AI는 제목 후보와 초안을 빠르게 만드는 데 쓴다. 하지만 업종의 말투, 실제 상품명, 가격, 이벤트 기간, 병원·법률처럼 민감한 표현은 사람이 확인해야 한다. AI 초안은 납품물이 아니라 중간 재료다.

## 🧾 의뢰 전 자료를 받아야 한다

블로그 원고 대행에서 가장 위험한 순간은 자료 없이 글을 시작할 때다. 의뢰인이 "대충 알아서 써주세요"라고 해도 그대로 시작하면 안 된다. 자료가 없으면 AI가 일반론을 채우고, 일반론은 어디서 본 듯한 글이 된다.

작업 전에 받을 자료는 아래처럼 정리해둔다.

| 받아야 할 자료 | 이유 |
| --- | --- |
| 업종명과 서비스명 | 글의 기본 맥락 |
| 반드시 넣어야 할 표현 | 누락 방지 |
| 피해야 할 표현 | 과장·금지 표현 방지 |
| 참고할 기존 글 | 원하는 톤 확인 |
| 독자 대상 | 초보자용인지 구매 직전용인지 구분 |
| 글의 목적 | 정보 제공, 예약 유도, 구매 유도 구분 |

이 자료를 받는 과정 자체가 실력이다. 질문을 잘하면 원고 품질이 올라간다. 질문을 안 하면 AI가 빈칸을 채운다. 프리랜서가 할 일은 빈칸을 멋있게 꾸미는 것이 아니라, 빈칸을 줄이는 것이다.

## 🧪 포트폴리오는 가짜 업종으로 만든다

처음부터 실제 고객 사례가 있을 수는 없다. 그래서 포트폴리오는 가짜 업종으로 만든다. 단, 허술한 가짜면 안 된다. 실제로 있을 법한 상황을 정하고 원고 전후 비교를 만든다.

| 샘플 업종 | 넓은 제목 | 좁힌 제목 |
| --- | --- | --- |
| 필라테스 | 필라테스 효과 | 퇴근 후 어깨가 굳는 직장인을 위한 초급 필라테스 루틴 |
| 반려동물 용품 | 좋은 강아지 샴푸 | 피부가 예민한 강아지 샴푸를 고를 때 확인할 성분 |
| 세무사 사무실 | 세금 상담 안내 | 프리랜서가 종합소득세 전에 정리해야 할 자료 |
| 전자책 판매 | 전자책 소개 | AI 부업 전자책 업데이트 로그를 봐야 하는 이유 |

![재택 업무용 노트북과 책상](./images/home-office.jpg)

_참고자료 사진: 재택 원고 대행 포트폴리오는 완성 원고보다 작업 과정을 보여주는 편이 좋다._

포트폴리오에는 완성 원고만 넣지 않는다. 제목 후보, 목차, 원고 일부, 수정 전후 문장, 작업 범위를 같이 보여준다. 의뢰인은 글솜씨만 보는 것이 아니라 이 사람이 어떻게 일하는지 본다.

## 🔍 검수는 문체보다 사실부터 본다

AI 원고의 가장 큰 문제는 문체보다 사실이다. 그럴듯한 문장 안에 틀린 정보가 섞일 수 있다. 그래서 검수 순서는 아래처럼 둔다.

| 검수 순서 | 확인할 것 |
| --- | --- |
| 1. 사실 | 상품명, 가격, 날짜, 지역, 자격, 금지 표현 |
| 2. 독자 | 누구에게 말하는 글인지 |
| 3. 구조 | 제목과 본문 흐름이 맞는지 |
| 4. 문체 | AI 티 나는 표현이 반복되는지 |
| 5. 수정 범위 | 의뢰인이 확인해야 할 부분이 표시됐는지 |

문체 검수에서는 "이를 통해", "중요합니다", "도움이 됩니다", "효과적입니다"가 반복되는지 본다. 모든 문장을 화려하게 바꿀 필요는 없다. 오히려 짧게 자르고 구체적인 행동으로 바꾸는 편이 낫다.

| AI 초안 느낌 | 고친 문장 |
| --- | --- |
| 고객 만족도를 높일 수 있습니다 | 후기에서 반복되는 질문을 본문 앞쪽에서 먼저 답합니다 |
| 효율적인 운영이 가능합니다 | 주 1회 업데이트할 항목만 따로 묶습니다 |
| 전문적인 이미지를 제공합니다 | 상담 전 확인할 자료를 표로 정리합니다 |

## 🔗 블로그 글로 연결한다

원고 대행은 그 자체로도 부업이지만, 애드센스 블로그 글감으로도 좋다. 작업하면서 배운 기준을 글로 남길 수 있기 때문이다.

- "AI 블로그 원고에서 지워야 할 표현"
- "의뢰 전에 받아야 할 자료 목록"
- "업종별 블로그 제목을 좁히는 방법"
- "원고 수정 범위를 계약서처럼 적는 법"

이런 글은 포트폴리오 역할도 한다. 의뢰인이 검색으로 들어와서 글을 읽고, 이 사람이 어떤 방식으로 원고를 다루는지 확인한다. 블로그가 단순 홍보 페이지가 아니라 작업 방식을 설명하는 공간이 된다.

AI 블로그 원고 대행은 집에서 가능한 일이 맞다. 하지만 아무 글이나 빠르게 뽑는 일이 아니다. 자료를 받고, 독자를 정하고, 목차를 만들고, 초안을 고치고, 수정 범위를 관리하는 일이다. 처음에는 작은 상품으로 시작하고, 포트폴리오 3개를 만든 뒤 실제 마켓 화면을 보면서 의뢰인이 어떤 이름으로 이 일을 찾는지 확인하는 편이 현실적이다.

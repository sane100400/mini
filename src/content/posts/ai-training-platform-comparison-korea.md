Outlier, DataAnnotation, Alignerr 같은 AI 트레이닝 플랫폼을 처음 보면 솔직히 혹한다. 집에서 AI 답변을 평가하고 돈을 받는 구조라니, 재택 부업으로는 꽤 좋아 보인다. 화면을 하나씩 보면 준비해야 할 것도 같이 보인다. 공개 일감, 심사, 언어, 전문 분야를 먼저 확인하면 기대치를 훨씬 현실적으로 잡을 수 있다.

내 기준에서는 영어 지시문을 읽을 수 있고, 코딩이나 글쓰기처럼 보여줄 실력이 있고, 일이 열릴 때까지 다른 작업을 같이 준비할 수 있는 사람에게 더 잘 맞는다. 한국어 작업만 기다리기보다 영어 작업 가능성, 평가 샘플, 포트폴리오를 함께 챙기는 쪽이 낫다.

| 내가 먼저 본 것 | 확인할 기준 |
| --- | --- |
| 공개 일감 | 지금 모집이나 기회가 보이는지 본다 |
| 언어 | 한국어와 영어 작업 가능성을 같이 본다 |
| 심사 | 테스트와 평가 과제 준비 범위를 본다 |
| 전문성 | 코딩, 법률, 의료, 수학 같은 분야가 있으면 선택지가 넓어진다 |
| 정산 | 시간, 환율, 입금 기록을 남길 표를 준비한다 |

## Outlier는 공개 기회부터 확인한다

상단 대표 이미지는 Outlier 공개 기회 화면을 직접 캡처했다. 2026년 6월 10일 캡처 기준으로 공개 기회 수가 0으로 표시됐다. 이 숫자 하나로 플랫폼 전체를 판단할 수는 없다. 다만 "가입하면 누구나 바로 시작한다"고 말하기는 어렵다.

| Outlier 확인 질문 | 내 기준 |
| --- | --- |
| 공개 기회가 보이는가 | 보이면 지원 경로와 조건 확인 |
| 한국어 작업이 있는가 | 계정 안 매칭까지 확인 필요 |
| 영어 작업이 가능한가 | 긴 지시문을 읽고 평가 가능한지 먼저 판단 |
| 전문 분야가 있는가 | 글쓰기, 코딩, 번역, 사실 검증을 나눠 보기 |

Outlier는 일감과 매칭을 함께 보는 편이 좋다. 가입 화면이 열려 있어도 실제 작업 배정까지는 시간이 걸릴 수 있다. 그 사이에는 블로그 글, 평가 샘플, 포트폴리오를 같이 준비하면 남는 것이 생긴다.

## DataAnnotation은 코딩 가능 여부에서 체감이 갈린다

DataAnnotation은 코딩 가능자에게 더 선명하게 보였다. 공식 페이지에는 코딩 전문가 지원과 시간당 단가 문구가 함께 표시된다. 비전공자가 무조건 불가능하다는 뜻은 아니지만, 코딩 작업을 노릴수록 준비 기준이 올라간다.

| 준비할 것 | 이유 |
| --- | --- |
| 영어 지시문 읽기 | 평가 기준과 작업 조건을 이해해야 함 |
| Python 또는 JavaScript 기본기 | 코딩 작업 후보가 더 분명해짐 |
| 작은 GitHub 프로젝트 | 문제 해결 과정을 보여주기 좋음 |
| 코드 설명 연습 | 모델 답변의 오류를 말로 설명해야 하는 경우가 있음 |

여기서 볼 지점은 "코드를 조금 만질 수 있다"와 "모델 답변의 오류를 설명할 수 있다"가 다르다는 것이다. 코딩 작업을 노린다면 지원서와 함께 작은 프로젝트와 README를 정리해두는 편이 좋다.

## Alignerr는 전공자형 부업처럼 느껴졌다

Alignerr 공개 홈 화면은 평균 단가, AI 인터뷰, 도메인별 전문가 모집 문구를 보여준다. 이 화면만 봐도 단순 반복 작업보다는 전문 분야 평가에 가까운 플랫폼이라는 인상을 준다.

| 맞을 가능성이 있는 사람 | 먼저 준비할 것 |
| --- | --- |
| 언어·번역 경험자 | 문장 평가 샘플 |
| 개발 경험자 | 코드 리뷰식 설명 |
| 수학·과학 전공자 | 풀이 과정 설명 |
| 법률·의료 등 전문 분야 경험자 | 자격, 경력, 표현 제한 확인 |

이런 플랫폼은 단순 반복 작업보다 전문 분야 평가에 가깝게 봐야 한다. 전공이나 업무 경험이 있으면 맞을 수 있고, 그 경험을 짧은 샘플로 보여줄 준비가 필요하다. 내게는 "남는 시간에 클릭하는 일"보다 "내 전문성을 짧게 증명하는 일"에 가까워 보였다.

## Mindrift는 역할별 난이도를 따로 봐야 한다

Mindrift 같은 플랫폼도 역할별로 나눠 보면 이해하기 쉽다. 같은 AI 트레이닝이라도 어떤 작업은 글을 읽고 비교하는 수준이고, 어떤 작업은 특정 분야 지식이 필요하다.

| 비교 항목 | 같이 볼 것 |
| --- | --- |
| 단가 | 심사 시간과 대기 시간을 함께 봐야 함 |
| 일감 | 열리는 시기가 불규칙함 |
| 언어 | 한국어만으로는 기회가 제한됨 |
| 전문성 | 분야별 테스트가 있음 |
| 정산 | 환율, 수수료, 세금 기록이 필요함 |

화면에서 시간당 단가가 높아 보여도 심사 시간이 길거나 일감이 불규칙하면 실제 월수입은 작게 끝나기도 한다. 반대로 단가가 낮아도 작업이 꾸준하고 자기 실력과 맞으면 연습용으로는 의미가 있다.

## 지원 전 준비물

| 순서 | 확인할 것 | 통과 기준 |
| --- | --- | --- |
| 1 | 공개 화면에서 모집이나 기회가 보이는지 | 현재 지원 가능한 경로가 있음 |
| 2 | 한국어 또는 영어 작업이 가능한지 | 지시문을 읽고 답변 가능 |
| 3 | 심사 방식과 필요한 전문성이 무엇인지 | 테스트 준비 범위 설명 가능 |
| 4 | 정산 방식과 기록 관리가 가능한지 | 작업 시간과 입금 내역을 남길 표가 있음 |
| 5 | 이 플랫폼 하나에 시간을 몰아도 되는지 | 대체 글감과 포트폴리오 작업이 같이 있음 |

대부분의 경우 답은 마지막에서 갈린다. 플랫폼 지원과 함께 남는 자산을 만들어두면 대기 시간도 덜 아깝다. AI 트레이닝 플랫폼 비교 과정 자체도 애드센스 블로그 글감이 된다.

지원 전에 만들 준비물은 아래처럼 잡는다.

| 준비물 | 쓰임 |
| --- | --- |
| 영어 자기소개 | 지원서와 프로필 |
| 작업 가능 분야 표 | 언어, 코딩, 글쓰기, 전문 분야 구분 |
| AI 답변 평가 샘플 | 심사 연습과 포트폴리오 |
| GitHub 또는 블로그 링크 | 결과물 증거 |
| 정산 기록표 | 시간, 환율, 입금 내역 관리 |

Outlier 같은 플랫폼은 확인해볼 만하다. 일이 오면 하고, 일이 없을 때는 블로그 글, 평가 샘플, GitHub 링크처럼 남는 증거를 만들면 된다. 재택 AI 부업은 하나의 플랫폼보다 여러 개의 작은 증거를 쌓는 쪽이 안정적이다.

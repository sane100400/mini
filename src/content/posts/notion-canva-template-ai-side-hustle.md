Notion이나 Canva 템플릿은 AI 부업 글에서 자주 나온다. 만들기는 쉽다. 문제는 팔리는 템플릿과 그냥 예쁜 템플릿이 다르다는 점이다. 특히 한국어 사용자를 대상으로 한다면 화면이 예쁜 것보다 실제 쓰는 상황이 먼저다.

예를 들어 "생산성 플래너"는 너무 넓다. 대신 "대학생 팀플 일정표", "1인 쇼핑몰 상품 업로드 체크리스트", "전자책 판매자 후기 관리표"처럼 좁히면 설명이 쉬워진다.

![AI 작업 화면과 책상이 보이는 홈 오피스](./images/ai-workspace.jpg)

## 템플릿은 기능보다 상황을 판다

처음 만드는 템플릿은 기능이 많을 필요가 없다. 기능이 많으면 설명서도 길어진다. 구매자는 템플릿을 산 뒤 바로 자기 일에 넣고 싶어 한다.

| 넓은 상품명 | 더 팔기 쉬운 상품명 |
|---|---|
| 일정 관리 템플릿 | 퇴근 후 부업 30일 체크표 |
| 독서 노트 | 전자책 제작 자료 정리표 |
| 콘텐츠 캘린더 | 블로그 글감 100개 관리표 |
| 가계부 | 프리랜서 수입 지출 기록표 |

이름이 구체적이면 상세페이지 문구도 쉬워진다. 누구를 위한 템플릿인지 설명하는 데 시간을 덜 쓴다.

![템플릿 내용을 확인하는 노트북 화면](./images/desk-laptop.jpg)

## AI는 초안과 예시 데이터를 만든다

AI를 쓰기 좋은 부분은 빈칸을 채우는 일이다. 템플릿 안에 예시 데이터가 없으면 구매자는 어떻게 써야 할지 모른다. 블로그 글감표라면 예시 제목 20개, 발행 상태, 카테고리, 다음 행동이 들어가야 한다.

Canva 템플릿도 마찬가지다. 빈 디자인만 올리는 것보다 한국어 문구가 들어간 샘플 5장을 보여주는 편이 낫다. 예시는 구매자가 자기 상품에 적용할 장면을 상상하게 만든다.

## 처음부터 마켓을 넓히지 않는다

처음 판매처는 하나만 잡아도 된다. 크몽, 오투잡, 블로그 문의, 스마트스토어 중 하나를 고른다. 판매처마다 상세페이지 말투와 이미지 규격이 달라서 한꺼번에 올리면 관리가 늦어진다.

![콘텐츠 제작 작업을 준비하는 책상](./images/content-desk.jpg)

초반 목표는 판매 100개가 아니다. 구매자가 어떤 문구에 반응하는지 확인한다. 그래서 템플릿은 1개를 크게 만드는 것보다 작은 템플릿 3개를 내보내는 편이 낫다.

한국어 템플릿은 아직 빈틈이 있다. 해외 템플릿을 그대로 번역하면 어색한 경우가 많고, 한국 플랫폼에서 쓰는 말과 맞지 않을 때도 있다. 그 차이를 줄이면 AI로 만든 템플릿도 단순 복붙 상품처럼 보이지 않는다.

## 템플릿 글에는 실제 마켓 화면이 필요하다

Notion·Canva 템플릿 판매를 말하면서 예쁜 스톡 사진만 넣으면 설득력이 약하다. 아래는 Notion 공식 템플릿 마켓플레이스 화면이다. 검색창, Work·School·Life 분류, 템플릿 카드가 보인다. 템플릿 판매 글에는 이런 실제 마켓 화면이 들어가야 한다.

![Notion 공식 템플릿 마켓플레이스 화면. 검색창과 카테고리, 템플릿 카드가 보인다.](./images/notion-templates-screen.jpg)

이 화면에서 봐야 할 것은 디자인이 아니다. 분류 방식이다. Notion은 Work, School, Life처럼 사용 장면으로 템플릿을 나눈다. 한국어 템플릿도 마찬가지다. "예쁜 플래너"보다 "퇴근 후 부업 30일 체크표", "전자책 업데이트 로그", "크몽 상품 관리표"처럼 사용 장면이 먼저다.

## 한국어 템플릿은 번역보다 맥락이 중요하다

해외 템플릿을 그대로 번역하면 말은 맞지만 쓰임이 어색해진다. 한국 사용자는 플랫폼 이름, 업무 흐름, 결제 방식, 판매 채널이 다르다. 예를 들어 Gumroad 중심 템플릿을 그대로 번역하면 크몽이나 블로그 문의 흐름과 맞지 않는다.

| 해외식 템플릿 | 한국어 사용 장면으로 바꾼 템플릿 |
|---|---|
| Creator dashboard | 전자책·블로그·크몽 부업 대시보드 |
| Content planner | 네이버 블로그·티스토리 발행 관리표 |
| Product launch checklist | 크몽 전자책 상품 등록 체크표 |
| Client tracker | 재택 프리랜서 문의·견적 관리표 |
| Finance tracker | 부업 수입·세금·환불 기록표 |

템플릿을 팔려면 기능보다 말이 먼저 맞아야 한다. 사용자가 자기 상황을 바로 떠올리면 설명이 짧아진다.

## AI로 템플릿을 만들 때 사람 손이 필요한 부분

AI는 템플릿 구조와 예시 데이터를 빠르게 만든다. 하지만 그대로 팔면 비슷해 보인다. 사람 손이 필요한 부분은 아래다.

| 작업 | 사람이 봐야 하는 이유 |
|---|---|
| 필드 이름 | 한국 사용자가 쓰는 말로 바꿔야 한다 |
| 예시 데이터 | 실제 부업 상황과 맞아야 한다 |
| 설명 문구 | 템플릿 사용 순서를 알려줘야 한다 |
| 빈칸 수 | 너무 많으면 처음 쓰기 어렵다 |
| 모바일 화면 | Notion은 모바일에서 표가 답답할 수 있다 |

예를 들어 "Status"를 "진행 상태"로 바꾸는 것만으로는 부족하다. 진행 상태 안에 "글감", "초안", "캡처 완료", "발행", "업데이트"처럼 실제 블로그 운영 단계를 넣어야 한다.

## Canva가 막힐 때도 템플릿 판매 글은 쓸 수 있다

Canva 공개 템플릿 화면은 자동화 캡처가 막혀 이번 글에는 넣지 않았다. 막힌 화면을 억지로 넣는 것보다, 정상적으로 캡처된 Notion 화면과 크몽 화면을 쓰는 편이 낫다. 대신 Canva 템플릿을 다룰 때는 어떤 산출물을 팔지 구체적으로 적는다.

| Canva 템플릿 | 한국어 판매 장면 |
|---|---|
| 카드뉴스 템플릿 | 전자책 홍보용 5장 카드뉴스 |
| 썸네일 템플릿 | 블로그·유튜브 쇼츠 썸네일 |
| 상세페이지 섹션 | 스마트스토어 첫 화면 문구 카드 |
| 후기 이미지 | 구매 후기 캡처 정리용 템플릿 |
| PDF 표지 | 전자책 표지와 목차 페이지 |

Canva 템플릿은 예쁜 디자인만으로 팔기 어렵다. 문구 샘플이 있어야 한다. "제목을 입력하세요"보다 "Outlier 지원 전 확인할 5가지"처럼 실제 문구가 들어간 샘플을 보여줘야 한다.

## 크몽에서 템플릿 수요를 확인한다

국내 판매처를 생각한다면 크몽 화면도 같이 본다. 전자책과 AI 서비스 카테고리가 보이는 화면은 템플릿 판매자에게도 힌트가 된다.

![크몽 공개 메인 화면. 전자책과 AI 서비스 카테고리에서 템플릿 상품 수요를 추정할 수 있다.](./images/kmong-marketplace-screen.jpg)

템플릿 상품명은 아래처럼 잡는다.

| 상품명 | 구성 |
|---|---|
| AI 부업 블로그 글감표 | 제목, 검색 의도, 이미지 출처, 발행일 |
| 전자책 업데이트 로그 | 수정 항목, 링크 교체, 독자 질문 |
| 크몽 상품 등록 체크표 | 제목, 설명, 가격, 수정 범위 |
| 숏폼 대본 관리표 | 후킹, 본문, CTA, 업종 |
| 바이브 코딩 외주 체크표 | 기능, 자료, 배포 링크, 검수 항목 |

이런 상품은 작은 가격으로 시작하기 좋다. 구매자는 템플릿 자체보다 자기 시간을 아끼는 구조를 산다.

## 템플릿 상세페이지에 넣을 것

템플릿 상세페이지는 아래 순서로 쓴다.

1. 누구를 위한 템플릿인지
2. 어떤 화면으로 구성되어 있는지
3. 예시 데이터가 들어 있는지
4. 모바일에서 쓸 수 있는지
5. 수정 방법이 쉬운지
6. 업데이트를 해주는지

여기서 "깔끔한 디자인"이라는 말은 뒤로 빼도 된다. 실제로 더 중요한 건 "구매 후 바로 자기 글감 10개를 넣을 수 있는가"다.

## 블로그 글로 이어지는 주제

템플릿 하나를 만들면 블로그 글도 같이 나온다.

- AI 부업 글감표에 꼭 넣을 항목
- 전자책 업데이트 로그 템플릿 만드는 법
- Notion 템플릿을 한국어 사용자에게 맞추는 기준
- Canva 카드뉴스 템플릿에 넣을 문구 샘플
- 크몽에서 팔 템플릿 상품명 정하는 법

템플릿 판매는 단독 부업으로도 가능하지만, 애드센스 블로그와 묶으면 더 좋다. 블로그는 사용법을 설명하고, 템플릿은 실행 파일이 된다.

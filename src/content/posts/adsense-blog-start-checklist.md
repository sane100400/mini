애드센스 블로그를 시작한다고 하면 보통 먼저 떠올리는 것은 글 개수다. 하지만 글 개수보다 먼저 정해야 하는 것은 블로그의 기준이다.

이 블로그는 단순 데이터라벨링, 해외 설문, 정체가 불분명한 앱테크를 메인 소재로 두지 않는다. 대신 Outlier 같은 AI 트레이닝 플랫폼, 바이브 코딩, 전자책, 블로그 운영처럼 한국인이 집에서 확인하고 실험할 만한 AI 부업을 기록한다.

![노트북으로 애드센스 블로그 주제를 정리하는 책상](./images/desk-laptop.jpg)

## 주제는 좁게 잡는다

큰 주제는 AI 부업이지만 모든 AI 부업을 다루지는 않는다.

- 애드센스 블로그 운영
- 전자책 판매와 템플릿 상품
- AI 숏폼 대본과 편집
- 블로그 원고 대행
- 스마트스토어 상세페이지 문구
- 프리랜서 플랫폼 프로필 만들기

주제를 좁히면 글감은 줄어든다. 대신 방문자가 이 블로그가 어떤 문제를 다루는지 빠르게 이해한다.

![블로그 글감과 키워드를 노트에 정리하는 모습](./images/writing-desk.jpg)

## 글은 실험 기록으로 쓴다

AI 초안을 그대로 올리는 방식은 오래가기 어렵다. 그래서 글 구조는 아래처럼 잡는다.

1. 하려는 실험
2. 사용한 도구와 조건
3. 실제 산출물
4. 들어간 시간
5. 막힌 부분
6. 다음에 수정할 점

방문자는 가능하다는 말보다 어디서 막히고 어떻게 고쳤는지를 보고 싶어 한다.

## 첫 목표

처음 목표는 월수익이 아니다.

- 글 30개 작성
- 카테고리 4개 유지
- 검색 유입이 생기는 글 파악
- 애드센스 승인 신청
- 문의 또는 전자책 클릭 발생 여부 확인

수익은 그 다음이다. 먼저 블로그가 특정 문제를 꾸준히 해결하는 사이트처럼 보여야 한다.

## 애드센스 블로그라면 AdSense 화면부터 확인한다

애드센스 글을 쓰면서 AdSense 화면이 하나도 없으면 글의 힘이 빠진다. 그래서 이 글부터는 스톡 사진 대신 공식 페이지 캡처를 넣는다. 아래 이미지는 Google AdSense 공식 시작 페이지다. 로고, 가입 버튼, 수익 창출 메시지가 한 화면에 같이 보인다. 이런 화면을 넣어야 독자가 "이 사람이 실제 서비스를 확인했구나"라고 느낀다.

![Google AdSense 공식 시작 페이지. 로고와 가입 버튼, 수익 창출 메시지가 보인다.](./images/google-adsense-start-screen.jpg)

애드센스 블로그를 시작하기 전에 이 화면에서 봐야 할 부분은 세 가지다. 첫째, AdSense는 광고 네트워크이지 글 자동 수익화 버튼이 아니다. 둘째, 가입 버튼이 보인다고 승인까지 자동으로 이어지지 않는다. 셋째, 콘텐츠 품질과 사이트 구조를 먼저 갖춰야 한다. 애드센스 글감이 부족한 블로그는 승인 전에도 막히고, 승인 후에도 조회수를 못 만든다.

내가 블로그를 새로 만들 때는 AdSense 신청 자체를 목표로 두지 않는다. 신청은 중간 단계다. 먼저 "광고가 붙어도 이상하지 않은 사이트인가"를 본다. 빈 카테고리, 얇은 글, 비슷한 제목, 출처 없는 주장, 스톡 이미지만 이어지는 글은 광고 전에 신뢰부터 약하다. 그래서 첫 30개 글은 승인용 글이 아니라 사이트의 기준을 만드는 글로 잡는다.

## 승인 전에 확인할 체크리스트

애드센스 승인 이야기는 검색하면 많이 나온다. 하지만 대부분 "글 몇 개 필요해요?"에서 멈춘다. 실제로는 글 개수보다 사이트가 독립된 주제를 갖고 있는지가 더 중요하다. 나는 아래 항목을 먼저 본다.

| 항목 | 최소 기준 | 확인 방법 |
|---|---|---|
| 주제 | AI 부업 안에서도 플랫폼, 블로그, 바이브 코딩처럼 묶임이 있어야 한다 | 카테고리마다 글 제목 5개 이상 적어본다 |
| 글 분량 | 얕은 소개글보다 실제 판단 기준이 있어야 한다 | 글마다 표, 사례, 캡처, 체크리스트를 넣는다 |
| 이미지 | 스톡 사진만 쓰지 않는다 | 공식 페이지나 내가 만든 샘플 화면을 섞는다 |
| 페이지 | 소개, 문의, 개인정보처리방침, RSS, sitemap이 있어야 한다 | 푸터와 robots.txt를 확인한다 |
| 내부 링크 | 한 글에서 다음 글로 넘어갈 이유가 있어야 한다 | 관련 글을 본문 중간에 연결한다 |

이 체크리스트를 통과하지 못하면 승인 신청을 미룬다. 신청을 빨리 넣는다고 손해는 크지 않지만, 떨어진 뒤 무엇을 고쳐야 할지 모르면 시간이 더 오래 걸린다. 특히 AI 부업 블로그는 비슷한 글이 많다. "AI로 돈 버는 법"이라는 넓은 말보다 "Outlier 공개 기회가 비어 있을 때 지원할지", "DataAnnotation 코딩은 어느 정도 실력이 필요한지", "v0로 만든 랜딩페이지를 외주로 팔 수 있는지"처럼 구체적인 질문이 필요하다.

## 첫 30개 글의 역할을 나눈다

처음부터 모든 글이 검색 유입을 가져오지는 않는다. 그래서 글을 역할별로 나눠야 한다. 어떤 글은 검색 유입용이고, 어떤 글은 신뢰를 쌓는 글이고, 어떤 글은 내부 링크를 받쳐주는 글이다.

| 글 역할 | 예시 제목 | 필요한 자료 |
|---|---|---|
| 검색 유입 | Outlier 같은 AI 부업 플랫폼 비교 | 공식 페이지 캡처, 플랫폼별 표 |
| 신뢰 형성 | AI 부업 포트폴리오 결과물 3개 | 샘플 캡처, 수정 전후 비교 |
| 수익 연결 | 전자책 업데이트 루틴 | Gumroad, 크몽, 판매 페이지 구조 |
| 도구 검토 | 바이브 코딩 부업은 작은 납품물부터 | v0, Cursor, GitHub 화면 |
| 운영 기록 | 30일 타임테이블 | 발행표, 내부 링크 계획 |

이렇게 역할을 나누면 글감이 덜 흔들린다. 애드센스 블로그는 글을 많이 올리는 게임처럼 보이지만, 실제로는 방문자가 다음 글을 눌러도 어색하지 않은 구조를 만드는 일이다. 같은 주제의 얇은 글 30개보다 역할이 다른 긴 글 10개가 더 낫다.

## 스톡 이미지를 줄이는 방식

스톡 이미지를 완전히 쓰지 말자는 뜻은 아니다. 책상 사진이나 노트북 사진은 분위기를 잡는 데 도움이 된다. 문제는 모든 글이 그런 사진으로 끝나는 경우다. 애드센스, Outlier, v0, CapCut, Notion 같은 주제를 다루면서 실제 화면이 없으면 글이 금방 가벼워진다.

그래서 앞으로 이미지는 세 종류로 나눠 넣는다.

| 이미지 종류 | 넣는 이유 | 예시 |
|---|---|---|
| 공식 페이지 캡처 | 실제 서비스를 확인했다는 근거 | AdSense, Outlier, CapCut |
| 직접 만든 샘플 | 내가 설명한 작업물을 보여줌 | 랜딩페이지, 계산기, 원고 전후 비교 |
| 분위기 사진 | 글 사이의 호흡을 조절 | 책상, 노트, 작업 화면 |

애드센스 글에는 AdSense 로고 화면이 들어가야 한다. 플랫폼 비교 글에는 플랫폼 화면이 들어가야 한다. 숏폼 글에는 CapCut이나 YouTube Shorts 화면이 들어가야 한다. 이렇게만 해도 글의 인상이 달라진다.

## 신청 전 마지막 점검

애드센스 신청 전에는 아래 순서로 본다.

1. 홈 첫 화면에 얇은 글만 노출되지 않는가
2. 최신 글 3개가 검색자가 실제로 궁금해할 소재인가
3. 글마다 최소 2개 이상의 실제 화면 또는 샘플 이미지가 있는가
4. 제목과 본문이 같은 말을 반복하지 않는가
5. 외부 공식 페이지 링크가 필요한 곳에 들어갔는가
6. sitemap과 robots.txt가 정상인가
7. 모바일에서 표와 이미지가 깨지지 않는가

여기까지 맞춘 뒤 신청한다. 애드센스는 블로그의 끝이 아니라 시작선이다. 광고가 붙은 뒤에도 글이 얕으면 수익은 거의 움직이지 않는다. 그래서 지금 이 블로그는 먼저 "검색자가 오래 읽을 글"을 만드는 쪽으로 방향을 바꾼다.
